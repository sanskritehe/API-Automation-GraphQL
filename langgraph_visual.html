LangGraph — How the Graph Works

State · Nodes · Edges · Conditional Routing · Retry Loop

1 · Core Concepts
🗂
StateGraph
The top-level object. You define a typed state dict and attach nodes + edges to it. Every node reads from and writes back to this shared state.
⬡
Node
A plain Python function (state) → patch. It receives the full state, does work (calls an LLM, etc.), and returns only the fields it wants to update.
→
Normal Edge
Always goes from node A to node B. Used for unconditional transitions like START → generate.
⋈
Conditional Edge
A router function reads the state and returns a string key ("pass" / "fail"). LangGraph maps that key to the next node.
🔁
Cycle
Unlike a DAG, LangGraph allows cycles. The fail edge loops back to generate, creating the retry loop. A counter in state stops infinite loops.
🏁
END
A built-in sentinel node. When an edge points to END the graph halts and returns the final state to the caller.
2 · The State Object (shared across all nodes)
# Defined once as a TypedDict — every node reads/writes fields from this
class AgentState(TypedDict):
  # ── inputs (set before graph starts) ─────────────────────────────────
  prompt : str # full prompt.md content (ticket + Confluence spec)
  app_dir : str # path to app/ folder to write generated files into

  # ── written by generate node ──────────────────────────────────────────
  generated_code : str # raw code block returned by Claude Sonnet

  # ── written by judge node ─────────────────────────────────────────────
  verdict : str # "PASS" or "FAIL"
  feedback : str # judge explanation — fed back into next generate call

  # ── managed by router ─────────────────────────────────────────────────
  iterations : int # incremented each time generate runs
  final_code : str # set when verdict == PASS — this is returned to pipeline.py

4 · Edge Definitions
FromToTypeCondition / Router logic
START generate normal Always. Kicks off the first generation.
generate judge normal Always. Every generation is judged.
judge router fn normal Always. Passes state to the router function.
router fn END conditional verdict == "PASS"
router fn generate conditional verdict == "FAIL" AND iterations < 3
router fn END conditional iterations ≥ 3 (max retries — use best attempt)
5 · How It Maps to Code
# ── 1. Define state ──────────────────────────────────────────────────────
class AgentState(TypedDict):
  prompt, app_dir, generated_code, verdict, feedback, iterations, final_code

# ── 2. Define nodes (plain functions) ────────────────────────────────────
def generate(state: AgentState) → dict:
  msgs = [{ "role": "user", "content": state["prompt"] + state.get("feedback", "") }]
  code = claude_sonnet(msgs)  # calls Anthropic API
  return { "generated_code": code, "iterations": state["iterations"] + 1 }

def judge(state: AgentState) → dict:
  result = claude_opus(judge_prompt + state["generated_code"])
  verdict, feedback = parse_verdict(result)
  return { "verdict": verdict, "feedback": feedback }

def router(state: AgentState) → str:  # returns edge key, NOT a state patch
  if state["verdict"] == "PASS":   return "pass"
  if state["iterations"] ≥ 3:     return "max"
  return "fail"

# ── 3. Build the graph ────────────────────────────────────────────────────
graph = StateGraph(AgentState)

graph.add_node("generate", generate)              # register nodes
graph.add_node("judge", judge)

graph.add_edge(START, "generate")                  # normal edges
graph.add_edge("generate", "judge")

graph.add_conditional_edges(                       # conditional edge
  "judge",                                       # source node
  router,                                        # router function
  { "pass": END, "fail": "generate", "max": END }  # key → next node map
)

app = graph.compile()                             # compile into runnable

# ── 4. Invoke from pipeline.py ───────────────────────────────────────────
final_state = await app.ainvoke({
  "prompt": prompt_content, "app_dir": app_dir,
  "iterations": 0, "feedback": ""
})
final_code = final_state["final_code"]              # handed to deploy_to_repo()
6 · State Evolution Through a FAIL → FAIL → PASS Run
START
prompt="GET /appointments/{id}..."   iterations=0   generated_code=None   verdict=None   feedback=""
generate ①
generated_code="def get_appointment(id):..."   iterations=1
judge ①
verdict="FAIL"   feedback="Missing 404 handling when appointment not found"
router → fail
iterations=1 < 3  →  edge key = "fail"  →  next node = generate
generate ②
generated_code="def get_appointment(id): ... raise HTTPException(404)..."   iterations=2   # feedback was appended to prompt
judge ②
verdict="PASS"   feedback="Correct. 404 handled, response schema matches spec."
router → pass
verdict="PASS"  →  edge key = "pass"  →  next node = END   final_code = generated_code
END
final_code returned to pipeline.py → deploy_to_repo()   2 iterations used, 1 remaining
